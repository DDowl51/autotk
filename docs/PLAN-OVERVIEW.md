## 跨需求的技术决策（提案，待确认）

- **仓库结构**（已有 `pnpm-workspace.yaml`，走 monorepo）：
  ```
  apps/mobile      ← 现在的 autotk（手机端）
  apps/desktop     ← 管理中心 Electron 应用（#1）：UI + 文件夹监视 + 本地 LAN 文件服务
  services/hub     ← #1 云协调中心（设备协调/状态/配置/任务路由 + 视频中转降级）
  services/license ← #5 激活 SaaS（独立、国内可访问）
  apps/web         ← #5 网页（客户看用量 + 管理员后台）
  packages/license-sdk ← 激活 SDK（autotk/未来产品接）
  packages/shared  ← 共享类型（AutomationParams 等，三端共用）
  ```
- **#1 拓扑**：手机不一定与操作员 PC 同局域网 → 需**云 Hub**(always-on) 让两边都连上；视频**同局域网直传、跨网经 Hub 中转降级**。Hub 与 #5 共用账号/部署。
- **#5（SaaS）**：独立云服务，Node+TS（Fastify/NestJS）+ PostgreSQL+Prisma；国内可访问。**通用、不耦合产品**；防破解走基线（设备绑定+心跳+远程封禁+签名+加固），另留**可选不透明 payload 通道**供产品自行加固。
- **网页前端**：React + Vite（#5 网页 + 管理员后台 + 可复用做 Hub 看板）。
- **账号体系**：#1（Hub）与 #5 **同一套**（已确认）。
